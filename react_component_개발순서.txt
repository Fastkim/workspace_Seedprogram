정적인 componet를 개발 ->
각 component에 어떤 데이터들이 있는지 파악 ->
state(component의 lifeCycle동안 데이터(값)가 변하는것)인 데이터를 구분,
state데이터는 다른데이터에 의해서 계산해서 만들어지면안됨.
component가 외부로부터 데이터를 받는다면 부모 component이다.

state는 갱신해야한다, state가 바뀔때마다 component를 갱신

형제componet간에 prop을 전달받고싶으면 부모에게 state를 보내면된다.

state의 2가지 조건
1. component의 lifeCycle동안 데이터(값)가 변하는것을 말한다.
2. properties가 아니여야한다.
3. 원천데이터는 state일만하다. ex) 1(state)+ 2(state) = 3(x)

원래는 모듈(기능)별로 component 파일을 따로 만든다.
